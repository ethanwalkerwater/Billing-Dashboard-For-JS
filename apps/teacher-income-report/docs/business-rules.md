# 薪资业务规则

本文档记录 `packages/billing-core/src/payroll-core.js` 已实现的当前口径。修改公式时，必须同步修改内核测试和本文档。

## 总公式

```text
课时反馈奖金 = 月度课时费 × 反馈系数
学员介绍提成 = 归属学员当月学费 × 20%
服务/管理奖金 = 服务学员当月学费 × 7%
Bonus 薪水 = 课时反馈奖金 + 介绍提成 + 服务奖金 - 基础薪水 × 1.2
个人总收入 = 基础薪水 + Bonus 薪水 + 补贴报销 - 个人五险 - 个税
公司总成本 = 基础薪水 + Bonus 薪水 + 补贴报销 + 公司五险
```

Bonus 按公式保留负数，页面会显示提醒，不自动归零。

## 课时费

- 优先使用课表中的 `课程总价格`。
- 金额缺失时，按 `课程单价 × 课程时长 × 取消比例` 回退计算。
- 取消比例与课时账单共享同一份 `CANCELLATION_RATES`。

## 反馈系数

| 老师类型 | 规则 |
|---|---|
| 全职 | 基础 47%；学习提升、责任心、个人魅力每有一项进入全职老师前 50%，增加 5%，最高 62% |
| 兼职 | 默认 60% |
| 缺少任一反馈分数 | 按兼职 60% 处理并给出提醒 |

前 50% 人数使用 `floor(可参与排名的全职老师数 × 0.5)`。边界分数并列时，同分老师都进入该指标的前 50%。

## 提成

- 学生姓名会去掉末尾编号，再与学生归属表匹配。
- 同一老师同时是归属老师和服务老师时，两项同时计入，合计 27%。
- 学生当月学费为负数时，提成基数按 0 处理，不生成负提成。

## 缺失数据

- 缺基础薪水：按 0 计算并提醒。
- 缺报销：按 0 计算并提醒。
- 缺五险个税：相应金额按 0 计算并提醒。
- 缺反馈分数：按兼职系数计算并提醒。

```json
{

```
