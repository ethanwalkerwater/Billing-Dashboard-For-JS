# 老师收入与公司成本产品更新计划

**目标：** 新增独立的「老师收入详情」app，作为现有 `billing-report` 课时费计算 app 的下游，按月计算每位老师的个人总收入与公司总成本，并保留可追溯的拆分明细。`billing-report` 继续只负责课时费计算。

**核心公式：**

```text
老师个人总收入 = 基础薪水 + Bonus 薪水 + 补贴报销 - 个人五险 - 个税
公司总成本 = 基础薪水 + Bonus 薪水 + 补贴报销 + 五险公司总支出部分
Bonus 薪水 = 月度课时费 * 学生反馈系数 + 学员介绍提成 + 服务/管理奖金 - 基础薪水 * 1.2
```

## 算法设计

### 1. 月度课时费

沿用现有逻辑：

```text
月度课时费 = 老师视图中该老师该月所有课程 amount 汇总
```

现有金额规则：

- 优先使用原始 CSV 的 `课程总价格`。
- 如果缺失，回退到 `课程单价 * 课程时长 * 取消比例`。
- 取消比例沿用现有 `CANCELLATION_RATES`。

### 2. 学生反馈系数

数据来源：

```text
apps/parent-report/data/teacher-score-averages.csv
```

已有字段：

```text
老师
学习提升效果
责任心与服务态度
个人魅力
```

计算规则：

```text
全职老师：
三项都进入全职老师前 50%：62%
两项进入全职老师前 50%：57%
一项进入全职老师前 50%：52%
零项进入全职老师前 50%：47%

兼职老师：
课时费系数固定为 60%，后续可修改参数。
```

实现建议：

- 只使用 `雇佣属性 = 全职` 的老师参与排名。
- 每个指标单独排名。
- 分数越高排名越靠前。
- 前 50% 用老师数量边界处理：例如 10 位老师时，第 5 名算前 50%，第 6 名不算。
- 如果人数为奇数，建议使用 `floor(全职老师人数 * 50%)` 作为入围人数；例如 9 位老师时前 4 名入围。这个是从“10 人第 5 名入围”的规则推导出的实现口径。
- 每位老师统计 3 个指标中进入前 50% 的数量。
- 全职老师系数 = `0.47 + 命中数量 * 0.05`，最高 `0.62`。
- 兼职老师不参与全职前 50% 排名，默认课时费系数为 `0.60`。
- 没有反馈分数的老师视为兼职老师，使用兼职课时费系数。
- 系数参数需要进入配置，不应写死在计算公式里。

建议配置结构：

```json
{
  "payrollParameters": {
    "fullTimeFeedbackBaseRate": 0.47,
    "fullTimeFeedbackStepRate": 0.05,
    "fullTimeFeedbackMaxRate": 0.62,
    "partTimeLessonRate": 0.6,
    "ownerCommissionRate": 0.2,
    "serviceCommissionRate": 0.07,
    "baseSalaryDeductionMultiplier": 1.2
  }
}
```

已确认：

- 前 50% 边界按名次处理：10 位老师时第 5 名入围，第 6 名不入围。
- 没有反馈分数就是兼职老师，不需要参与全职反馈排名。

待确认：

- 分数并列时是否都算进入前 50%。
- 只有部分指标缺失时，是缺失项算未入围，还是整体系数按兼职处理。
- 兼职老师是否参与学员介绍提成和服务/管理奖金，目前第一版先不考虑。

### 3. 学员介绍提成

计算：

```text
学员介绍提成 = sum(学生当月学费 * 20%)，按学生的 归属老师(20%) 归属
```

学生当月学费来源：

```text
现有学生视图 result.totals.amount
```

匹配规则：

- 优先用 `学生姓名` 匹配课表里的 `学生`。
- 课表学生可能带联系方式后缀，例如 `学生名-101`，需要复用现有家长报告里的清洗规则。
- 多学生拆分课表行时，需要确认金额是否整行归属每个学生，还是按学生人数平分。现有系统学生视图目前会让每个学生都看到该行金额。
- 如果同一老师同时是学生归属老师和服务老师，需要同时生成两条提成明细，合计 27%。

### 4. 服务/管理奖金

计算：

```text
服务奖金 = sum(学生当月学费 * 7%)，按学生的 服务老师(7%) 归属
```

命名建议：

- 公式里写的是 `学员介绍提成+管理奖金`。
- 数据源字段实际是 `服务老师(7%)`。
- 产品 UI 建议叫 `服务/管理奖金`，明细里标注来源字段为 `服务老师(7%)`。

### 5. Bonus 薪水

计算：

```text
全职课时反馈奖金 = 月度课时费 * 学生反馈系数
兼职课时反馈奖金 = 月度课时费 * 兼职课时费系数，默认 60%
提成奖金 = 学员介绍提成 + 服务/管理奖金
基础薪水扣减 = 基础薪水 * 1.2
Bonus 薪水 = 课时反馈奖金 + 提成奖金 - 基础薪水扣减
```

待确认：

- Bonus 是否允许为负数。建议第一版按公式保留负数，因为它会影响总收入和公司成本；如果业务希望不低于 0，需要明确为 `max(Bonus, 0)`。
- 兼职老师如果没有基础薪水，`基础薪水 * 1.2` 是否为 0。建议缺失基础薪水时按 0 计算，同时给出数据提醒。

### 6. 个人总收入与公司总成本

```text
个人总收入 = 基础薪水 + Bonus 薪水 + 补贴报销 - 个人五险 - 个税
公司总成本 = 基础薪水 + Bonus 薪水 + 补贴报销 + 公司五险支出
```

展示时必须保留拆分：

```text
基础薪水
月度课时费
学生反馈系数
课时反馈奖金
学员介绍提成
服务/管理奖金
基础薪水 * 1.2 扣减
Bonus 薪水
补贴报销
个人五险
个税
个人总收入
公司五险支出
公司总成本
```

## 已确认业务规则

1. 前 50% 排名边界：10 位老师时第 5 名算前 50%，第 6 名不算。

2. 没有反馈分数的老师按兼职老师处理，不参与全职反馈排名。

3. 兼职老师课时费系数默认为 60%，后续需要支持参数修改。

4. 同一个老师既是学生归属老师又是服务老师时，同时获得 20% + 7%。

5. 当前个税直接读取 `个税.csv` 的结果；后续开发老师收入产品时，个税需要单独设计。

6. 五险截图仍然有效。

7. 兼职老师是否参与 20% 学员介绍提成和 7% 服务/管理奖金，目前第一版先不考虑。

8. 后续每月常规输入为课程表 CSV、五险+个税 xlsx/csv、补贴报销 xlsx/csv。

9. 老师基础薪水表和学生归属服务表作为项目默认主数据维护，默认值来自本次提供的文件；网页端提供上传入口用于后续调整或本次覆盖。
